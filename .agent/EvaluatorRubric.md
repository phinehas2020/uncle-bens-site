# Evaluator Rubric

Grade every milestone on a 1-5 scale:

## product_depth
- 5: Milestone meaningfully satisfies the contract with real behavior
- 3: Partial — some items implemented, others stubbed
- 1: Display-only or placeholder

## functionality
- 5: All user flows and edge cases work
- 3: Happy path works, edge cases broken
- 1: Core flows broken

## design_quality
- 5: UI is coherent, intentional, polished
- 3: Functional but default-looking
- 1: Broken layout or unusable

## code_quality
- 5: Readable, consistent, verified
- 3: Works but messy
- 1: Fragile or untested

## Hard fail conditions
- Critical behavior broken or missing
- Required verification was skipped
- Implementation is display-only where real behavior was required
